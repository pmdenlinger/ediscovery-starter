# Review Coding Standards (Example)

**Core fields:**
- Responsiveness: Responsive / Non‑Responsive
- Confidentiality: Public / Confidential / Highly Confidential
- Issue Tags: [Define issues relevant to matter]
- Privilege: Privileged / Non‑Privileged / Work Product
- Redaction Reason: PII / Privilege / Trade Secret

**Consistency rules:**
- Families: keep consistent coding across family (unless protocol dictates otherwise)
- Email threads: prefer coding the inclusive email, but verify attachments
- Redactions: apply reason codes consistently; burn-in at production
