# EDRM Lifecycle (Concise Practitioner View)

1. Identification – scoping custodians, systems, data sources; risk & preservation candidates.
2. Preservation – legal hold notices; hold tracking; suspension of auto‑deletion; CoC start.
3. Collection – targeted, repeatable, well‑logged; device & cloud sources; forensically sound where needed.
4. Processing – inventory, deNIST, dedupe (global/custodian/family), timezone normalization, metadata integrity.
5. Review – batching, coding (responsive/non‑responsive/issue/privilege), QC (sampling, conflicts, consistency).
6. Analysis (parallel) – search strategy, threading, near‑dedupe, concept clusters, TAR if applicable.
7. Production – format selection (TIFF/PDF/native), redactions, placeholders, load files (DAT/OPT), delivery log.
8. Presentation – trial/hearings/expert display packages.

**Defensibility:** document _who/what/when/how_ at each phase (see CoC and QC).
